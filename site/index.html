Real example

New user: question to decision in 10 minutes

You have a question about your architecture. Grainulator suggests wheat, you init a sprint, and 10 minutes later you have a brief.

$ npx @grainulator/wheat init Question: Should we migrate from REST to GraphQL? Audience: Backend team Constraints: Must support existing mobile clients Sprint created. claims.json + wheat.config.json ready. wheat> /research "GraphQL performance and migration cost" r001 [factual|documented] GraphQL reduces mobile payload 15-25% r002 [risk|documented] REST caching (CDN) does not transfer to GraphQL r003 [estimate|web] Migration effort: 3-5 weeks for core endpoints wheat> /challenge r001 x001 [risk|documented] Payload reduction varies by endpoint; some REST endpoints are already optimized wheat> /brief Compiled 8 claims (1 conflict resolved) Written: output/brief.html Recommendation: Proceed with GraphQL for new endpoints only. Migrate existing REST endpoints opportunistically.
The ecosystem

Eight tools. Each does one thing.

Use what you need. Start with wheat. Everything else is optional.

w
wheat
Research engine
Ask a question, grow claims from multiple angles, compile a decision brief. The core of everything.
npx @grainulator/wheat init
farmer
Permission dashboard
Approve AI agent tool calls from your phone, another tab, or a meeting. Real-time WebSocket dashboard.
npx @grainulator/farmer start
g
grove
Shared tools
Sprint detection, manifest generation, HTML templates. The shared utilities the ecosystem relies on.
npx @grainulator/grove detect-sprints
m
mill
Export & publish
Turn compiled research into PDFs, CSVs, static sites. Every format you need to share decisions.
mill export --format pdf brief.html
s
silo
Knowledge packs
Reusable claim libraries. HIPAA constraints, migration risks, architecture patterns. Pull, don't re-research.
silo pull compliance --into claims.json
h
harvest
Analytics
Cross-sprint learning. Track prediction accuracy, find systemic blind spots, detect stale claims.
harvest analyze ./sprints/
f
field
Orchestration
Multi-sprint coordination. Dependencies, conflict detection, team assignment. The org-wide view.
field status

Philosophy

Built on four principles

The decisions behind the tools.

Satisficing over maximizing

You will never have perfect information. The goal is enough evidence to make a defensible decision, not a perfect one.

Claims over opinions

Every finding is a typed claim with an evidence tier and provenance. "I think" becomes a traceable claim ID.

Adversarial pressure

Comfortable agreement is the enemy of good decisions. Challenge, witness, and stress-test every claim.

Zero dependencies

Node built-ins only. No supply chain anxiety. No left-pad. The tools ship what they need and nothing more.

The name comes last. You build the crop, the steward, the orchard, the mill, the silo, the harvest, the field -- and only then do you name the machine.
